# 海龟战法（Turtle Trading System）在线回测系统 — 规格说明书

## 1. 项目概述

构建一个**Web在线回测平台**，用户可在浏览器中输入任意A股标的代码，系统自动拉取最新日线数据，运行海龟战法回测，并在前端展示完整的收益/风险/交易分析报告。支持数据每日更新，确保始终使用最新行情。

### 核心特性
- **在线交互**：Web界面，无需安装任何软件
- **任意标的**：输入任何A股代码即可回测
- **实时数据**：通过 Tushare API 拉取，支持每日更新
- **可视化报告**：交互式图表（权益曲线、回撤、交易标记等）
- **参数可调**：策略参数、回测区间均可在界面配置

## 3. 策略核心逻辑（A股简化版，仅做多）

海龟战法保留两套子系统：

| 参数 | 系统1（短线） | 系统2（长线） |
|------|:------------:|:------------:|
| 入场突破周期 | 20日 | 55日 |
| 出场突破周期 | 10日 | 20日 |
| 上次突破过滤 | 是 | 否 |

### 3.1 入场规则
- 收盘价 **突破** 过去 N 日最高价 → 买入1单位
- 系统1：若上次突破最终盈利，跳过本次信号（防假突破）
- 加仓：每上涨 0.5×ATR 追加1单位，最多4单位

### 3.2 出场规则
- 止损：入场价 - 2×ATR，触及即平该单位
- 趋势结束：收盘价跌破过去 M 日最低价 → 全平

### 3.3 头寸规模
- 1 Unit = (账户权益 × 1%) / ATR
- ATR 周期：20日

## 4. API 接口设计

### 4.1 股票搜索
```
GET /api/search?q=平安
→ { results: [{code:"000001", name:"平安银行", market:"SZ"}, ...] }
```

### 4.2 数据拉取/更新
```
POST /api/fetch_data
Body: { ts_code: "000001.SZ", force_refresh: false }
→ { status: "ok", data_range: "2010-01-04 ~ 2026-07-09", rows: 4012, cached: true|false }
```
- 优先读本地缓存，若缓存不存在或 `force_refresh=true` 则从 Tushare 拉取
- 缓存按 `{ts_code}.csv` 存储在 `task4/cache/` 目录

### 4.3 数据状态检查
```
POST /api/data_status
Body: { ts_code: "000001.SZ" }
→ { exists: true, last_date: "2026-07-09", is_fresh: true, rows: 4012 }
```
- 判断数据是否覆盖到最新交易日

### 4.4 执行回测
```
POST /api/backtest
Body: {
  ts_code: "000001.SZ",
  system: 1,           // 1=短线(20/10), 2=长线(55/20)
  start_date: "2018-01-01",
  end_date: "2024-12-31",
  initial_capital: 1000000,
  commission: 0.0003,
  slippage: 0.0001,
  atr_period: 20,
  risk_percent: 0.01,
  max_units: 4
}
→ { report: { metrics, trades, equity_curve, benchmark_curve } }
```

## 8. 数据更新机制

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  用户点击     │────▶│  检查缓存     │────▶│  缓存存在    │
│ 「更新数据」  │     │  cache/       │     │  且日期新鲜  │
└─────────────┘     └──────────────┘     └──────┬──────┘
                                                 │
                              ┌──────────────────┤
                              ▼                  ▼
                       ┌──────────┐      ┌──────────────┐
                       │ 缓存不存在 │      │ 缓存过期      │
                       │ 或强制刷新 │      │ (非最新交易日) │
                       └────┬─────┘      └──────┬───────┘
                            │                   │
                            ▼                   ▼
                       ┌──────────────────────────────┐
                       │  调用 Tushare API 拉取全量     │
                       │  daily(ts_code, start, end)   │
                       │  写入 cache/{ts_code}.csv     │
                       └──────────────────────────────┘
```

- **新鲜度判定**：对比缓存文件最新日期与 Tushare 交易日历，若缓存最新日期 < 最近交易日 → 标记为过期
- **增量更新**：仅拉取缓存缺失部分（从最后日期到最新交易日），追加写入
- **前端指示**：数据状态灯（绿色=新鲜，黄色=可更新，红色=无缓存）
